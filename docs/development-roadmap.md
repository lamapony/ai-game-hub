# Долгосрочная дорожная карта DIMAS fest

Дата: 2026-07-01

## Северная звезда

DIMAS fest должен стать форматом живой AI-вечеринки, которую можно запустить где угодно: парк, двор, фестиваль, корпоратив, лагерь. Один ведущий открывает комнату, люди подключаются телефонами, AI помогает создавать задания, судить, озвучивать и превращать телефоны в распределённую сцену.

Главная ценность: не “ещё один квиз”, а совместное физическое действие в реальном пространстве, где AI усиливает ведущего и делает каждый раунд уникальным.

## Продуктовые принципы

- **Мгновенный старт:** ведущий должен создать комнату и начать игру за 2 минуты.
- **Телефон как контроллер:** игроку не нужна регистрация, приложение, аккаунт или инструкция.
- **AI как персонаж, а не форма:** AI должен звучать как ведущий/дух/режиссёр, а не как чат-бот.
- **Физическое действие важнее экрана:** лучшие игры заставляют людей двигаться, записывать, искать, кричать, играть сценки.
- **Надёжность важнее вау:** если AI или сеть упали, вечер не должен развалиться.
- **Ведущий контролирует темп:** AI предлагает и озвучивает, но host всегда может пропустить, переиграть, остановить.

## Горизонт 1: playable production

Цель: провести полноценный вечер без ручного вмешательства разработчика.

Критерии готовности:

- Production deploy на Cloudflare Workers.
- Отдельный production Supabase проект.
- Все secrets настроены в GitHub Actions.
- Host может создать комнату, запустить 3 игры, поставить раунд на паузу, пропустить фазу,
  перезапустить игру и вернуться в hub.
- 8-20 игроков проходят вечер на телефонах.
- Минимум 1 дополнительный speaker-telephone стабильно подключается и играет.
- AI failure states показываются host-у понятным текстом.
- Старые комнаты и uploads можно чистить без ручного SQL.

Ключевые задачи:

- Проверить cleanup job на production после первого deploy: dry-run, затем реальное удаление старых комнат.
- Расширить fallback-задания и fallback-судейство после первого field test.
- Проверить host controls на field test: pause, skip, restart round, force back to hub.
- Проверить сообщения ошибок для камеры, микрофона, upload и AI на iOS/Android field test.
- Провести field test на реальных телефонах и сети.

## Горизонт 2: reliable party engine

Цель: превратить текущий набор игр в устойчивый движок комнат, игр и раундов.

Критерии готовности:

- Единый game registry вместо ручной логики в host/player routes.
- Явные state transitions с проверками допустимых фаз.
- Server-authoritative updates для критичных действий.
- Базовый automated regression pack в CI.
- Базовая observability для AI latency и cleanup/API failures.

Ключевые задачи:

- Вынести общий game contract: `HostView`, `PlayerView`, `initialState`, `actions`, `score`.
- Добавить tests для `src/lib/room.ts` и game state transitions.
- Расширить structured logging на room lifecycle и upload failures.
- Расширить retry/backoff на остальные Supabase writes после field test.
- Добавить optimistic UI только там, где state можно безопасно откатить.

## Горизонт 3: stronger games

Цель: сделать игры повторяемо смешными, понятными и разными по динамике.

Направления:

- **Soundscape Battle 2.0:** пресеты темпа, “режиссёрский монтаж”, replay лучших моментов, manual speaker calibration.
- **Challenge 2.0:** задания по типам: сценка, пантомима, коллективный звук, физический объект, дуэль.
- **Photo Hunt 2.0:** AI-комментарии по каждому кадру, галерея вечера, приз зрительских симпатий.
- **Новые игры:** Prompt Battle, Park Detective, Human Statue, Object Auction, Ritual Builder.

Критерии готовности:

- Каждая игра объясняется игроку в 1 экран.
- Раунд длится предсказуемо и не провисает.
- У каждой игры есть fallback без AI.
- У каждой игры есть “смешной момент” на host screen.

## Горизонт 4: reusable event product

Цель: можно проводить разные события без правки кода.

Ключевые возможности:

- Настройка темы события: название, стиль ведущего, язык, возрастной режим.
- Наборы игр: короткий вечер, большой фестиваль, семейный режим, корпоратив.
- Admin/event dashboard: список комнат, live status, storage usage.
- Экспорт итогов: галерея, победители, лучшие AI-комментарии.
- Custom voices и safe content filters.

Критерии готовности:

- Новый event можно создать из dashboard.
- Host получает ссылку и QR без разработчика.
- Все тексты и AI prompts завязаны на event profile.
- Есть базовая модерация и режим “без жести”.

## Горизонт 5: platform

Цель: DIMAS fest становится платформой для живых AI-игр.

Возможности:

- Плагинная модель игр.
- Marketplace/internal library сценариев.
- Поддержка нескольких host screens.
- Поддержка нескольких зон/локаций на фестивале.
- Интеграции с ticketing/event tools.
- Платные event packs.

Это не нужно строить до доказанного повторного использования на реальных событиях.

## Метрики

Продуктовые:

- Время от открытия `/` до первого раунда.
- Доля игроков, успешно подключившихся с первого раза.
- Доля раундов, завершённых без ручного вмешательства.
- Средняя длительность паузы между раундами.
- Количество “смешных моментов” за вечер по оценке host-а.

Технические:

- AI latency p50/p95 по типам вызовов.
- Upload failure rate.
- Realtime reconnect rate.
- Speaker cue drift и missed cue count.
- Ошибки camera/mic permission по устройствам.

Операционные:

- Стоимость AI на один вечер.
- Storage на один вечер.
- Количество ручных действий перед деплоем.
- Время восстановления после падения AI-провайдера.

## Решения, которые пока не принимаем

- Не строим аккаунты и платежи до повторных живых мероприятий.
- Не делаем сложный редактор игр до появления хотя бы 5-7 стабильных игр.
- Не оптимизируем bundle size ценой скорости разработки, пока он не влияет на полевой запуск.
- Не строим multi-tenant admin, пока production event можно обслужить простым Supabase проектом.

## Ближайшие 30 дней

1. Закрыть production secrets и первый Cloudflare deploy.
2. Провести smoke test с 2-3 телефонами.
3. Провести field test с 8+ игроками.
4. Зафиксировать top 10 friction points после field test.
5. Проверить fallback-задания и host controls на field test.
6. Проверить cleanup старых комнат/uploads на production.
7. Подготовить “event profile” как простой config, без dashboard.

4. Добавить 1-2 новые игры, не требующие тяжёлого media pipeline.
5. Добавить галерею итогов вечера.
6. Добавить basic observability и cost tracking.
