## Что строим в V1

**Каркас Jackbox** (host + phones через realtime) + **одна убийственная игра — Soundscape Battle с 5 колонками**. Всё на Lovable Cloud (БД + realtime) и Lovable AI Gateway (LLM, vision, картинки, TTS, STT). Дизайн — тёмный zinc + park-gradient как в прототипе.

## Архитектура — Jackbox-style

**Две роли, одно приложение:**

- **Host (`/host`)** — открывается на ноуте ведущего, подключён к **главной колонке**. Показывает большой экран: код комнаты `ПАРК-7421`, лидерборд, текущую игру, инструкции, генерируемые AI картинки, говорит голосом через TTS.
- **Player (`/play`)** — открывается на телефоне. Игрок вводит код + имя + выбирает команду. Дальше — кнопки/камера/микрофон под текущую игру.
- **Speaker (`/speaker?slot=2..5`)** — открывается на 4 дополнительных телефонах, каждый подключён по Bluetooth к одной из 4 «парковых» колонок (Дух Дуба, Ветер, Белка, Эхо). Получает realtime-команды от host и играет audio (TTS-фразы, soundscape-сэмплы). Это и есть «AI как живой персонаж парка».

**Realtime через Lovable Cloud:**

- Таблицы: `rooms`, `teams`, `players`, `rounds`, `submissions`, `votes`, `speaker_cues`.
- Supabase Realtime channels: один канал на комнату. Host публикует state, телефоны игроков слушают. Speaker-устройства слушают отдельный `speaker_cues` стрим.
- Без auth: вход по `room_code` + `player_id` в localStorage (анонимные сессии). RLS-политики разрешают читать/писать только в комнату, чей код есть в запросе.

## Soundscape Battle — flow (главная фича)

1. **Host:** жмёт «Запустить Soundscape», AI генерит 3 темы («Ссора белок на рассвете», «Дискотека грибов», «Лес во время апокалипсиса»), голосование на телефонах за тему.
2. **Запись (3 мин):** на каждом телефоне игрока — большая красная кнопка «Запись». Команды бегают по парку, ловят звуки (птицы, шорох, смех, удары палкой). Записи (≤15 сек, mp3) аплоадятся в Cloud Storage.
3. **AI-микс:** для каждой команды LLM (Gemini 3 Flash) получает Whisper-транскрипции/описания записей + тему, строит «партитуру» — JSON-расписание: какой клип в какую секунду играть на какой колонке (slot 1–5) + какие TTS-фразы вставлять между ними (Дух Дуба комментирует). Это и есть «вау»: звук реально движется по парку через 5 устройств.
4. **Воспроизведение (по 60 сек на команду):** host рассылает `speaker_cues` с таймстампами. Все 5 устройств воспроизводят синхронно (общее `startAt` + `setTimeout` на каждое cue). Главная колонка ещё и зачитывает интро от AI-ведущего.
5. **Голосование:** на телефонах игроков — 3 кнопки «🔥 Атмосфера / 😂 Угар / 🎨 Креатив». Свою команду выбрать нельзя.
6. **Очки:** AI-судья (Gemini) даёт фидбек-текст + бонусные очки за креатив, голоса конвертируются в основные очки. Host показывает результат + анимацию.

**Почему это вау:** ни у кого нет 5-канального пространственного AI-звука в парке. Это и есть та новая штука из плана.

## AI-вызовы (всё через серверные TanStack функции, `LOVABLE_API_KEY` server-side)


| Что                                             | Endpoint                                      | Модель                                        |
| ----------------------------------------------- | --------------------------------------------- | --------------------------------------------- |
| Темы, фидбек, режиссура микса                   | server fn `generateTopics`, `judgeSoundscape` | `google/gemini-3-flash-preview` (JSON output) |
| Расшифровка записей                             | server fn `transcribeAudio`                   | `openai/gpt-4o-mini-transcribe`               |
| Голос AI-ведущего                               | server fn `speak` (стримит mp3)               | `openai/gpt-4o-mini-tts`                      |
| Трофей-картинка финала (на следующих итерациях) | server route `/api/generate-image` (SSE)      | `openai/gpt-image-2`                          |

## Что меняется по сравнению с прототипом

- **Убираем:** случайные очки через `Math.random()`, alert-моки, ручной выбор «моей команды» на каждом телефоне без серверной правды.
- **Сохраняем:** визуальный язык (dark zinc + emerald park gradient, Space Grotesk + Inter, rounded-3xl карточки), названия команд (Лесные/Озёрные/Древесные/Полевые).

## Технические детали

**Стек:** TanStack Start (есть), Lovable Cloud (включаем), AI Gateway (включаем). Tailwind токены — park-зелёный gradient + zinc-950 base заводим в `src/styles.css` через oklch, без хардкода.

**Файлы (главное):**

```
src/routes/index.tsx              — лендинг: «Создать комнату» / «Присоединиться»
src/routes/host.$code.tsx         — экран ведущего
src/routes/play.tsx               — ввод кода + имя
src/routes/play.$code.tsx         — экран игрока (роутится на текущую игру)
src/routes/speaker.$code.tsx      — speaker-устройство (slot=N в query)
src/routes/api/speak.ts           — TTS стрим
src/games/soundscape/{Host,Player}.tsx
src/lib/{room,realtime,speakers}.ts
src/lib/ai/{gateway.server,topics.functions,judge.functions,transcribe.functions}.ts
supabase/migrations/0001_rooms.sql — таблицы + GRANTы + RLS
```

**Колонки и Bluetooth — как ведущий это разруливает:** на странице `/host` есть быстрый чек-лист: «Колонка 1 (главная) — этот ноут. Колонки 2–5 — открой `/speaker/ПАРК-7421?slot=2` на 4 телефонах, каждый подключён к своей Bluetooth-колонке». Тест-кнопка «Бухнуть звук на колонку N» в инструкции.

**Риски и mitigation:**

- Latency между speaker-устройствами: используем серверный timestamp + `startAt = now + 2s` буфер, чтобы все успели зашедулить cues.
- WiFi в парке: записи сжимаем (16 кГц моно WAV), TTS прегенерим заранее когда возможно, fallback на browser `SpeechSynthesis` если AI-call упал.
- Очки и сабмиты — server-authoritative, клиент только триггерит.
