# Project Notes for AI Agents (Cursor Composer, Codex, etc.)

This project is maintained directly in GitHub and deployed independently. Avoid rewriting published history unless explicitly requested.

## Current Focus: Grill + Bar Party Upgrade (Nørrebro)

**Master specification:** `docs/grill-bar-party-upgrade.md`

This spec was generated by Fable 5 as a complete TЗ for adding support for a new party format:
- 🔥 Grill phase (outdoor at Grønningen Nordvest, 17:00) — food, fire, active.
- 🍸 Bar phase (Viggos Bar, 20:00) — drinks, indoor, witty adult social games.

**Key rules (non-negotiable):**
- Nothing is removed. Existing games (soundscape, challenge, phototunt, etc.) stay fully working.
- Only supplement + adapt.
- Environment is the star (fire/smoke/tongs vs glasses/light/toasts).
- Adult, sharp, slightly sarcastic wit. No corporate safe language.
- Prompts must use strict JSON + few-shots + "+5 for using the environment" rubric (see spec section 3).

## How to work with this spec (Composer 2.5 / agents)

1. Read the full `docs/grill-bar-party-upgrade.md` first.
2. Note the adaptation requirement: The spec was written against an old standalone HTML. Current architecture uses:
   - `src/games/[name]/` folders (HostView.tsx + PlayerView.tsx pattern)
   - Supabase for state, artifacts, realtime, storage
   - Central rules in `src/lib/game-rules.ts`
   - Existing soundscape is the most advanced reference (spatial audio, recording, mixing)

3. Work **atomically**:
   - Foundation first (phase system)
   - One new game at a time
   - Follow section 4 "Атомарные инструкции" but translate the UI patterns to React/TSX + Supabase

4. After changes:
   - `bun run lint && bunx tsc --noEmit`
   - Test the specific game flow locally
   - Update game rules and any shared components

See also: `docs/composer-grill-bar-tasks.md` for prioritized atomic tasks.

## General agent rules
- Prefer adding new game folders over modifying core.
- All AI interactions should be prompt-driven with the style from the spec.
- Keep backward compatibility for "classic" park-style play if needed.
